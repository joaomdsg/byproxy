# Additional mandate — full-system familiarization (A/B: identical for every arm)

vialite is an unfamiliar framework and your fixes must be idiomatic to
it. Before you change any code, become fully conversant with the ENTIRE
system: every Go source file in the repository and every documentation
page under `docs/*.md` (23 pages — actions and lifecycle, scope,
distributed state, compositions, examples, and the rest). Several pages
document behaviors that silently constrain correct fixes; do not skip
any. Your final report must include a FRAMEWORK section citing, for each
fix you make, the doc page and the framework mechanism that makes the
fix idiomatic.
